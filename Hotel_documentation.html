Hotel Room Booking Management System

Industry-level backend CRUD application using FastAPI, PostgreSQL, SQLAlchemy ORM and Pydantic Models for managing hotel room bookings, customer reservations and stay records digitally.

View Complete Repository

Problem Statement

Develop a backend Hotel Room Booking Management System using FastAPI, PostgreSQL, SQLAlchemy ORM and Pydantic Models.

The system should perform complete CRUD operations for managing hotel booking records digitally.

The application should store and manage:

• Booking ID
• Customer Name
• Room Type
• Number Of Days

Project Objective

Build scalable backend APIs capable of:

• Adding hotel booking records
• Viewing booking details
• Updating room information
• Deleting booking records

The system should implement Response Models, Status Codes, Exception Handling, Dependency Injection and Middleware while storing data in PostgreSQL database.

Functional Requirements

  • Add hotel booking records using POST API
  • Fetch all booking records using GET API
  • Update room type and stay duration using PUT API
  • Delete booking records using DELETE API
  • Implement Exception Handling
  • Implement Dependency Injection
  • Implement Middleware Logging

Technical Requirements

FastAPI PostgreSQL SQLAlchemy ORM Pydantic Response Models Status Codes Exception Handling Dependency Injection Middleware Uvicorn
